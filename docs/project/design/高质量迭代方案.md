# 电影推荐系统高质量迭代方案

## 一、项目最终定位

本项目的正式方向是电影推荐系统。空气质量预测计划书属于历史备选题，不参与最终提交。

项目一句话定位：

> 构建一个融合用户评分行为与电影内容特征的混合推荐系统，并通过主实验、消融实验、用户分群、流行度偏置、显著性检验和交互式展示，系统评估不同推荐策略在准确性、覆盖率、多样性和可解释性之间的权衡。

## 二、核心研究问题

RQ1：不同推荐算法在 Top-N 电影推荐任务上的表现有何差异？

RQ2：内容特征是否能改善推荐准确性、覆盖率和多样性之间的平衡？

RQ3：混合推荐是否能缓解低活跃用户或新用户的冷启动问题？

RQ4：高准确率模型是否存在更强的热门电影偏置？

RQ5：推荐解释是否能增强系统展示效果和结果可信度？

## 三、正式实验矩阵

### 3.1 主实验

比较以下模型：

| 模型 | 作用 |
|---|---|
| Popularity | 最基础热门推荐基线 |
| UserCF | 基于相似用户的协同过滤 |
| ItemCF | 基于相似电影的协同过滤 |
| Matrix Factorization | 隐因子推荐模型 |
| Content-based | 基于电影类型、标题、标签、年份等内容特征 |
| Hybrid | 融合协同过滤、内容相似度和流行度 |

核心指标：

- Precision@K
- Recall@K
- NDCG@K
- MAP@K
- MRR@K
- Coverage
- Diversity

### 3.2 消融实验

消融设置用于回答“融合到底带来了什么”：

| 设置 | 解释 |
|---|---|
| popularity_only | 只使用热门程度 |
| cf_only_item | 只使用物品协同过滤 |
| content_only | 只使用内容相似度 |
| cf_content_no_pop | 去掉流行度项 |
| cf_pop_no_content | 去掉内容项 |
| content_pop_no_cf | 去掉协同过滤项 |
| hybrid_default | 默认融合权重 |

报告中应重点解释：某个模型即使不是所有准确率指标最高，也可能在覆盖率、多样性或偏置方面更优。

### 3.3 用户分群实验

将用户按训练集中评分数量分为：

- low_activity
- medium_activity
- high_activity

该实验用于分析冷启动或少交互用户场景。若 Content-based 或 Hybrid 在低活跃用户上更稳定，可以作为项目亮点。

### 3.4 流行度偏置分析

分析不同模型是否过度推荐热门电影。建议结合两个角度写：

1. 准确率高不等于推荐质量全面更好。
2. 推荐系统需要在命中用户兴趣和发现长尾内容之间权衡。

### 3.5 显著性与稳定性检查

新增正式实验产物：

- `significance_results.*`：基于每个用户 NDCG@K 的配对显著性检验。
- `cross_validation_results.*`：k-fold 结果均值和标准差。

这两类结果能支撑“实验可信度”，避免报告只依赖一次划分的偶然结果。

## 四、推荐正式运行命令

正式报告建议使用：

```powershell
python scripts/run_formal_experiment.py --data-dir data/raw/ml-latest-small --k 10 --max-eval-users 300 --min-user-ratings 5 --min-movie-ratings 5 --cv-folds 3
```

若机器性能或数据下载受限，可使用：

```powershell
python scripts/run_formal_experiment.py --sample --k 5 --max-eval-users 20 --cv-folds 3
```

但最终报告应尽量使用真实 MovieLens 数据，而不是 sample 数据。

## 五、报告写作主线

建议报告按以下逻辑展开：

1. 推荐系统中的信息过载和冷启动问题。
2. MovieLens 数据、评分矩阵稀疏性和内容特征构建。
3. 六类推荐算法设计。
4. 统一 Top-N 评价协议。
5. 主实验比较不同算法。
6. 消融实验分析内容、协同过滤和流行度的贡献。
7. 用户分群分析低活跃用户场景。
8. 覆盖率、多样性和流行度偏置分析。
9. 推荐解释与 Streamlit 系统展示。
10. 局限性与未来工作。

## 六、答辩讲法

答辩时不要说“我们实现了很多模型”，而应说：

> 我们围绕电影推荐中的准确性、多样性、流行度偏置和冷启动问题，构建了一个可复现实验框架。通过主实验、消融实验、分群实验和偏置诊断，我们发现不同推荐策略各有取舍；混合推荐的价值不只是追求单一最高分，而是在多个推荐质量维度之间取得更平衡的表现。

现场演示建议顺序：

1. 展示数据画像和模型对比。
2. 选择一个已有用户，切换不同模型，观察推荐结果差异。
3. 展示推荐理由。
4. 切到冷启动页面，输入类型偏好或关键词。
5. 展示消融、用户分群、偏置分析和显著性结果。
