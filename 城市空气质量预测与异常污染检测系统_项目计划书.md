## 二、项目背景与研究意义

空气污染会影响居民健康、城市交通、公共管理和环境治理。PM2.5、PM10、NO2、O3 等污染物浓度通常受到天气、季节、城市活动强度和突发事件等因素影响，具有明显的时间趋势、周期性和异常波动特征。

本项目拟融合空气质量、天气和日历特征等多源数据，构建一个城市空气质量预测与异常污染检测系统。项目将围绕一个主城市、一个核心污染物和一套完整可运行 Pipeline 展开，在此基础上实现多模型对比、异常检测和 Streamlit 交互展示，并视进度扩展到多城市、多污染物和深度学习模型。

## 三、研究问题

1. 天气因素，如温度、湿度、风速、降水，是否会影响 PM2.5 等污染物浓度？
2. 不同类型预测模型在空气质量预测任务上的表现有何差异？
3. 异常检测算法能否识别污染物浓度异常升高的时间点？
4. 如何通过交互式 Dashboard 展示空气质量趋势、预测结果、异常点和模型对比结果？

## 四、数据来源与数据融合方案

### 4.1 数据来源

1. 空气质量数据：OpenAQ API  
   获取目标城市监测站点的 PM2.5、PM10、NO2、O3 等污染物浓度数据。核心版本优先选择 PM2.5 作为预测目标，保证任务聚焦。

2. 天气数据：NOAA Climate Data Online API 或其他公开天气数据  
   获取目标城市对应时间段的温度、降水、风速、湿度等天气变量。

3. 日历与节假日数据：Python 日期库或公开节假日数据  
   构造星期、月份、季节、是否周末、是否节假日等时间特征。

4. 备选城市与时间范围：  
   优先选择数据完整度较高的纽约、洛杉矶或伦敦。核心版本使用 2023 年至 2025 年的日级数据；若数据缺失较多，则缩短时间范围或更换城市。

### 4.2 数据融合方案

本项目将数据统一到日级时间粒度，以空气质量数据为主表，天气数据和日历特征作为外部特征，通过日期字段和城市/站点字段进行连接。融合后的建模表包含：

- 时间特征：日期、星期、月份、季节、是否周末、是否节假日
- 污染物特征：PM2.5、PM10、NO2、O3 等，核心目标为 PM2.5
- 天气特征：温度、湿度、风速、降水
- 滞后特征：前 1 天、前 3 天、前 7 天污染物浓度
- 目标变量：下一天或未来 7 天 PM2.5 浓度
- 异常标记：基于统计阈值或异常检测算法得到的异常点

## 五、技术路线

1. 数据获取：编写 API 调用脚本，下载空气质量和天气数据，并保存为本地 CSV/Parquet 缓存。
2. 数据清洗：处理缺失值、重复记录、单位差异、异常极值和日期格式问题。
3. 数据融合：按日期合并空气质量、天气和日历特征，生成统一建模数据表。
4. 探索性分析：分析 PM2.5 趋势、季节性、天气相关性和异常峰值。
5. 预测建模：实现至少 5 种模型，包括基线模型、ARIMA/Prophet、Random Forest、XGBoost 和 LSTM 或替代模型。
6. 异常检测：实现统计阈值、Isolation Forest、LOF、One-Class SVM 等方法。
7. 模型评估：使用统一指标比较预测模型和异常检测方法，并进行误差分析。
8. 系统展示：使用 Streamlit 构建交互式 Dashboard。
9. 交付整理：完成项目报告、PPT、README 和演示材料。

## 六、算法设计与实验方案

### 6.1 预测模型

| 类型 | 算法 | 是否核心必做 | 作用 |
|---|---|---|---|
| 基线模型 | Naive Forecast / Moving Average | 是 | 提供最低性能参考 |
| 统计模型 | ARIMA 或 SARIMA | 是 | 建模趋势和周期性 |
| 时间序列模型 | Prophet | 是 | 建模趋势、季节性和节假日效应 |
| 机器学习 | Random Forest | 是 | 利用天气和滞后特征进行预测 |
| 机器学习 | XGBoost | 是 | 作为强基线模型进行对比 |
| 深度学习 | LSTM | 选做 | 若时间允许，用于建模长期依赖 |

### 6.2 异常检测模型

| 算法 | 是否核心必做 | 说明 |
|---|---|---|
| Z-score / IQR | 是 | 统计阈值方法，作为异常检测基线 |
| Isolation Forest | 是 | 基于随机切分的无监督异常检测 |
| LOF | 是 | 基于局部密度的异常检测 |
| One-Class SVM | 选做 | 学习正常样本边界，识别异常点 |

### 6.3 实验设计

预测任务采用按时间顺序划分训练集、验证集和测试集，避免未来信息泄漏。异常检测任务采用统计异常点、模型异常分数和典型案例分析结合的方式进行验证。所有模型使用统一数据集、统一指标和统一实验记录表，保证对比公平。

## 七、评估指标

预测任务指标：

- MAE：平均绝对误差
- RMSE：均方根误差
- MAPE：平均绝对百分比误差
- R²：拟合优度

异常检测指标：

- Precision、Recall、F1-score
- 异常点数量与时间分布
- 典型异常案例解释

工程交付指标：

- 主流程是否可一键运行
- Dashboard 是否支持交互展示
- README 是否包含环境配置、运行方式和模块说明
- GitHub/Gitee 是否具有清晰 commit 历史

## 八、交互式成果展示设计

计划使用 Streamlit 开发轻量级 Dashboard，核心页面包括：

1. 数据总览页：展示时间范围、记录数量、缺失情况和基础统计信息。
2. 趋势分析页：展示 PM2.5 历史趋势、季节变化和天气相关性。
3. 预测结果页：展示真实值与不同模型预测值的对比曲线。
4. 异常检测页：在时间序列图中高亮异常污染点，并展示异常原因分析。
5. 模型对比页：展示 MAE、RMSE、MAPE、F1 等指标排行榜。
