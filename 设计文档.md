# 数据挖掘期末项目计划书

## 项目名称

**融合协同过滤与内容特征的电影推荐系统：多算法对比、可解释分析与交互式演示**

英文题目：

**A Hybrid Movie Recommendation System with Collaborative Filtering and Content Features: Comparative Evaluation, Explainable Analysis, and Interactive Demonstration**

---

## 一、项目定位

本项目拟以电影推荐为应用场景，构建一个端到端的数据挖掘系统。项目将融合用户评分行为数据、电影元数据、电影类型、关键词、简介文本等多源信息，系统比较多种推荐算法在 Top-N 推荐任务中的表现，并进一步构建轻量级交互式推荐界面，用于展示推荐结果、模型对比和推荐解释。

本项目的定位不是简单实现一个推荐算法，而是模拟一篇应用型数据挖掘 workshop paper 的完整工作流程，包括：

1. 明确的数据挖掘问题；
2. 多源数据融合；
3. 多算法系统性对比；
4. 可复现实验流程；
5. 推荐结果可解释分析；
6. 可交互系统演示；
7. 学术论文式项目报告与答辩展示。

---

## 二、选题背景与研究意义

随着在线视频平台、流媒体服务和内容社区的发展，用户面对的电影、剧集和短视频数量迅速增长。如何从海量内容中为用户推荐符合其兴趣的电影，是推荐系统中的经典问题，也是数据挖掘在真实场景中的重要应用。

传统推荐方法主要包括基于流行度的推荐、协同过滤、矩阵分解和基于内容的推荐等。然而，不同方法在数据稀疏性、冷启动、可解释性和推荐多样性方面存在明显差异。例如，协同过滤能够捕捉用户行为模式，但对新用户或新电影较为敏感；基于内容的推荐能够利用电影类型、关键词和简介文本，但可能缺乏对群体偏好的建模能力。因此，如何融合用户行为数据与电影内容特征，并系统比较不同推荐算法的效果，是一个具有代表性的数据挖掘问题。

本项目希望围绕电影推荐任务，完成从数据处理、特征构建、模型训练、实验评估到交互展示的完整流程。最终成果不仅包括推荐模型本身，还包括模型对比分析、错误案例分析、推荐解释和可运行的推荐系统 Demo。

## 三、研究问题

本项目围绕以下研究问题展开：

### RQ1：不同类型推荐算法在电影推荐任务中的表现有何差异？

本项目将比较流行度推荐、基于用户的协同过滤、基于物品的协同过滤、矩阵分解、基于内容的推荐以及混合推荐方法，分析不同算法在准确性、覆盖率和推荐多样性方面的差异。

### RQ2：融合电影内容特征是否能够提升推荐效果？

本项目将引入电影类型、标签、关键词、简介文本等内容信息，比较仅使用用户评分行为与融合内容特征后的推荐性能差异。

### RQ3：混合推荐方法能否缓解冷启动问题？

对于新用户或评分记录较少的用户，协同过滤方法往往表现不稳定。本项目将设计基于用户偏好输入的冷启动推荐方案，分析内容特征在冷启动场景下的作用。

### RQ4：推荐结果是否具有可解释性？

本项目将尝试从电影类型匹配、相似电影、用户历史偏好、关键词相似度等角度解释推荐结果，使推荐系统不仅能给出结果，还能说明推荐原因。

## 四、数据来源与数据说明

本项目计划使用以下公开数据集：

### 1. MovieLens 用户评分数据

MovieLens 是推荐系统研究中广泛使用的公开数据集，包含用户对电影的评分记录。主要字段包括：

* userId：用户编号；
* movieId：电影编号；
* rating：用户评分；

该数据集将作为协同过滤、矩阵分解和推荐评估的核心数据来源。

### 2. MovieLens 电影元数据

MovieLens 数据中通常还包含电影基础信息，例如：

* movieId：电影编号；
* title：电影标题；
* genres：电影类型。

该数据将用于构建基于内容的推荐特征。

### 3. TMDB / IMDb 补充电影信息

为了增强内容特征，本项目可进一步引入 TMDB 或 IMDb 相关数据，例如：

* 电影简介；
* 关键词；
* 演员；
* 导演；
* 发行年份；
* 受欢迎程度；
* 平均评分；
* 电影海报链接。

这部分数据将用于扩展内容特征，提高推荐解释性和系统展示效果。

## 五、任务定义

本项目主要研究 Top-N 电影推荐任务。

给定用户历史评分行为和电影内容信息，系统需要为目标用户生成一个推荐电影列表：

* 输入：用户历史评分记录、电影内容特征、用户偏好选择；
* 输出：Top-N 推荐电影列表；
* 目标：推荐用户可能感兴趣且尚未观看或评分过的电影。

项目将同时考虑两类场景：

### 1. 历史用户推荐

对于已有评分记录的用户，系统根据其历史评分行为和相似用户/相似电影生成推荐结果。

### 2. 新用户冷启动推荐

对于没有历史评分记录的新用户，系统根据用户选择的偏好类型、喜欢的电影或关键词生成初始推荐结果。

## 七、算法设计

本项目计划实现并比较以下算法。

### 1. Popularity Baseline

基于电影流行度进行推荐，例如根据评分数量、平均评分或加权评分排序。该方法作为最基础的 baseline，用于衡量复杂算法是否真正带来提升。

### 2. User-Based Collaborative Filtering

基于用户之间的相似度进行推荐。若两个用户在历史评分上表现相似，则可以将一个用户喜欢的电影推荐给另一个用户。

主要步骤包括：

1. 构建用户-电影评分矩阵；
2. 计算用户相似度；
3. 选择 Top-K 相似用户；
4. 根据相似用户评分预测目标用户对未看电影的兴趣；
5. 生成 Top-N 推荐列表。

### 3. Item-Based Collaborative Filtering

基于电影之间的相似度进行推荐。若目标用户喜欢某些电影，则系统推荐与这些电影相似的其他电影。

主要步骤包括：

1. 构建电影-用户评分矩阵；
2. 计算电影之间的相似度；
3. 根据用户历史高评分电影寻找相似电影；
4. 聚合相似度得分；
5. 生成推荐列表。

### 4. Matrix Factorization

矩阵分解方法将用户和电影映射到低维隐向量空间，通过用户向量与电影向量的内积预测用户兴趣。

本项目计划采用 SVD 或 Surprise 库中的矩阵分解方法。该方法能够捕捉用户和电影之间的潜在语义关系，是推荐系统中的经典方法。

### 5. Content-Based Recommendation

基于电影内容特征进行推荐。该方法将电影类型、关键词、简介文本等转化为向量表示，然后计算电影之间的内容相似度。

可使用的特征包括：

* 电影类型 one-hot 编码；
* 关键词 TF-IDF 特征；
* 简介文本 TF-IDF 特征；
* 发行年份等数值特征。

对于用户历史喜欢的电影，系统计算其内容画像，再推荐与用户画像相似的电影。

### 6. Hybrid Recommendation

混合推荐方法综合协同过滤得分和内容相似度得分，得到最终推荐分数。

初步设计如下：

Final Score = α × Collaborative Filtering Score + β × Content Similarity Score + γ × Popularity Score

其中 α、β、γ 为权重参数。项目将通过实验比较不同权重组合下的推荐效果。

## 八、实验设计

### 1. 数据划分

项目将采用训练集、验证集和测试集划分方式。为了模拟真实推荐场景，可采用按用户划分或按时间划分的方法：

* 训练集：用于模型训练；
* 验证集：用于调参；
* 测试集：用于最终评估。

对于每个用户，保留部分高评分电影作为测试目标，检验模型是否能将这些电影排在推荐列表前列。

### 2. 评价指标

本项目计划使用以下推荐系统评价指标：

#### Precision@K

衡量推荐列表前 K 个电影中有多少是用户真正感兴趣的电影。

#### Recall@K

衡量用户真正感兴趣的电影中有多少被系统成功推荐出来。

#### NDCG@K

考虑推荐结果的排序位置，越靠前推荐到相关电影，得分越高。

#### MAP@K

衡量推荐列表整体排序质量。

#### Coverage

衡量系统能够推荐到的电影范围，避免模型只推荐少数热门电影。

#### Diversity

衡量推荐列表内部的多样性，避免推荐结果过于单一。

### 3. 对比实验

本项目将比较以下模型：

| 模型编号 | 模型名称                               | 类型    |
| ---- | ---------------------------------- | ----- |
| M1   | Popularity Baseline                | 基线方法  |
| M2   | User-Based Collaborative Filtering | 协同过滤  |
| M3   | Item-Based Collaborative Filtering | 协同过滤  |
| M4   | Matrix Factorization               | 隐因子模型 |
| M5   | Content-Based Recommendation       | 内容推荐  |
| M6   | Hybrid Recommendation              | 混合推荐  |

### 4. 消融实验

为了分析不同数据源和特征的贡献，本项目计划进行消融实验：

| 实验编号 | 实验设置        | 目的            |
| ---- | ----------- | ------------- |
| A1   | 仅使用用户评分数据   | 检验行为数据效果      |
| A2   | 仅使用电影类型特征   | 检验基础内容特征效果    |
| A3   | 加入关键词和简介文本  | 检验文本特征贡献      |
| A4   | 协同过滤 + 内容特征 | 检验多源融合效果      |
| A5   | 去除流行度项      | 分析流行度对推荐结果的影响 |

### 5. 冷启动实验

项目将模拟新用户冷启动场景：

1. 用户没有历史评分；
2. 用户选择喜欢的电影类型；
3. 用户选择 3–5 部喜欢的电影；
4. 系统根据内容相似度和流行度生成推荐结果。

该实验用于说明内容特征在冷启动场景下的价值。

## 九、结果分析计划

项目最终不仅展示模型分数，还将进行多角度分析。

### 1. 主结果分析

使用表格和柱状图展示各模型在 Precision@K、Recall@K、NDCG@K 等指标上的表现，分析不同算法的优缺点。

### 2. 用户群体分析

将用户按评分数量划分为：

* 低活跃用户；
* 中等活跃用户；
* 高活跃用户。

分析不同模型在不同用户群体上的表现差异。

### 3. 推荐多样性分析

比较不同算法推荐结果中电影类型的分布，分析模型是否存在过度推荐热门电影的问题。

### 4. 错误案例分析

选取若干推荐失败案例，分析模型为何推荐了用户不感兴趣的电影。可能原因包括：

* 用户兴趣过于多样；
* 历史评分数量过少；
* 电影类型标签过粗；
* 热门电影对推荐结果产生偏置；
* 文本特征无法准确表达电影风格。

### 5. 推荐解释分析

对于每个推荐结果，尝试给出解释，例如：

* 因为用户喜欢同类型电影；
* 因为该电影与用户高评分电影相似；
* 因为相似用户也喜欢该电影；
* 因为该电影在相同题材中评分较高。

## 十、系统设计与交互展示

本项目计划使用 Streamlit 构建轻量级交互式推荐系统。

### 1. 系统功能

系统主要包括以下模块：

#### 用户选择模块

用户可以选择：

* 已有用户 ID；
* 喜欢的电影类型；
* 喜欢的电影；
* 推荐电影数量 Top-K。

#### 推荐结果模块

系统输出：

* 推荐电影标题；
* 电影类型；
* 预测得分；
* 推荐理由；
* 海报或基础信息。

#### 模型对比模块

展示不同算法的评估结果，包括：

* Precision@K；
* Recall@K；
* NDCG@K；
* Coverage；
* Diversity。

#### 可视化分析模块

展示：

* 用户兴趣画像；
* 推荐结果类型分布；
* 模型性能对比图；
* 特征或内容相似度解释；
* 热门电影偏置分析。

### 2. 系统价值

该交互系统能够增强项目展示效果，使答辩不只是展示静态表格，而是可以现场演示：

1. 输入用户偏好；
2. 选择推荐算法；
3. 生成推荐列表；
4. 查看推荐理由；
5. 对比不同模型效果。

## 十五、项目亮点

本项目预计形成以下亮点：

### 1. 多源数据融合

项目不仅使用用户评分数据，还融合电影类型、关键词、简介文本等内容信息，体现多源数据融合的数据挖掘思想。

### 2. 多算法系统性对比

项目将实现并比较至少 5 种推荐算法，包括 baseline、协同过滤、矩阵分解、内容推荐和混合推荐。

### 3. 消融实验与冷启动分析

项目将分析不同数据源和特征对推荐性能的贡献，并模拟新用户冷启动场景，体现实验深度。

### 4. 可解释推荐

项目不仅输出推荐结果，还尝试解释推荐原因，提高推荐系统的透明度。

### 5. 交互式系统展示

通过 Streamlit 构建演示界面，使最终成果可运行、可展示、可复现。

### 6. 论文式报告组织

项目报告将采用数据挖掘 workshop paper 风格，强调研究问题、方法设计、实验分析和讨论总结。

## 十六、风险分析与应对策略

### 风险 1：外部电影元数据匹配困难

应对策略：

优先使用 MovieLens 自带的电影类型信息完成基础内容推荐。若 TMDB 或 IMDb 数据匹配顺利，则作为增强特征加入；若匹配成本过高，则不影响主流程完成。

### 风险 2：深度模型效果不稳定

应对策略：

本项目不将复杂深度学习模型作为核心依赖，而以协同过滤、矩阵分解和内容推荐作为主要方法。若时间允许，可尝试 Neural Collaborative Filtering 作为扩展实验。

### 风险 3：推荐指标实现复杂

应对策略：

优先完成 Precision@K、Recall@K 和 NDCG@K 三个核心指标，再扩展 Coverage 和 Diversity。

### 风险 4：系统开发时间不足

应对策略：

Streamlit 系统采用轻量级实现，优先保证核心功能，包括用户选择、推荐结果展示和模型对比图。高级交互功能作为可选扩展。

### 风险 5：团队协作不均衡

应对策略：

从第一周开始建立 Git 仓库和任务看板，每位成员负责明确文件和模块，最终报告中用 commit 记录、代码文件和章节署名证明贡献。

## 十七、预期结论

本项目预期能够得到以下结论：

1. 相比简单流行度推荐，协同过滤和矩阵分解方法在已有用户推荐场景下表现更好；
2. 基于内容的推荐在新用户或评分较少用户场景下具有优势；
3. 混合推荐方法可能在准确性、覆盖率和冷启动适应性之间取得更好平衡；
4. 推荐结果的可解释性能够提升系统展示效果和用户理解度；
5. 多算法对比和多源数据融合能够形成更完整、更有说服力的数据挖掘项目成果。
