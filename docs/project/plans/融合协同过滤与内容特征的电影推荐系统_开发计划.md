# 融合协同过滤与内容特征的电影推荐系统开发计划

## 一、开发目标

本开发计划以《融合协同过滤与内容特征的电影推荐系统项目计划书》为基础，面向 3 周项目周期，明确系统开发范围、任务拆解、人员分工、阶段里程碑、验收标准和风险控制方案。

项目最终目标是完成一个可运行、可复现、可展示的电影推荐系统。系统需支持 MovieLens 数据处理、多算法推荐、Top-N 评估、冷启动推荐、推荐解释和 Streamlit 交互展示，并形成项目报告、答辩 PPT 和团队分工声明。

## 二、开发范围与优先级

| 优先级 | 内容 | 说明 |
|---|---|---|
| P0 必须完成 | 数据处理、特征工程、M1-M6 推荐算法、评估指标、Streamlit 基础演示、README、报告和 PPT | 项目核心交付内容 |
| P1 重点完成 | 冷启动推荐、推荐解释、消融实验、覆盖率和多样性分析 | 支撑研究问题和答辩亮点 |
| P2 视进度完成 | TMDb/IMDb 外部元数据、电影海报展示、更大规模数据集、学术海报 | 不影响主流程验收 |

## 八、Git 协作规范

1. 每位成员至少每 2 天提交一次有效 commit。
2. commit 信息采用格式：`模块: 动作说明`，例如 `models: add item-based cf`。
3. 主要分支建议为 `main`，个人开发使用 `feature/data`、`feature/models`、`feature/app` 等分支。
4. 合并前需确认主流程可运行，避免提交破坏性代码。
5. 每周末进行一次代码整合和运行检查。

## 十、风险与应对

| 风险 | 影响 | 应对方案 |
|---|---|---|
| 数据规模过大 | 模型训练和调试缓慢 | 优先使用 MovieLens latest-small 或 1M，确保主流程完整 |
| 外部元数据匹配困难 | 内容特征和展示效果受限 | 外部元数据作为 P2，核心特征使用 MovieLens 类型和标签 |
| 推荐评估实现复杂 | 模型对比延误 | 先完成 Precision@K、Recall@K、NDCG@K，再扩展 MAP@K、Coverage、Diversity |
| Streamlit 开发时间不足 | 系统演示不完整 | 优先完成用户选择、推荐展示、模型对比三类核心页面 |
| 模型效果不稳定 | 报告结论不明显 | 保留 Baseline 与多模型对比，重点分析不同模型适用场景 |
| 团队协作冲突 | 代码整合困难 | 提前确定接口格式，每周至少一次整合测试 |

### 12.5 风险与边界

- 当前模型使用 dense user-item matrix，真实大规模数据需通过 `--max-ratings`、`--max-users`、`--max-movies` 和 `--max-matrix-cells` 控制规模。
- P2 外部元数据、海报、TMDb/IMDb 扩展仍暂不纳入本轮，避免引入 API Key、网络稳定性和展示复杂度风险。
- 本轮重点交付实验分析结果，为项目报告和答辩提供可引用表格与结论。
